# Quantus QA Automation Rules

## 역할
Figma 디자인을 분석하여 QA 테스트 케이스를 자동 생성하고 xlsx 파일로 저장하는 자동화 도구.

---

## 아키텍처

```
TC_Automation/
├── generate_tc.py        # xlsx 생성 엔진 (openpyxl 기반)
├── tc_data/<화면명>.py   # 화면별 TC 데이터 (screen_name + tc_data 변수)
└── output/               # 생성된 QA_[화면명]_TC.xlsx 저장 위치
```

**엔진/데이터 분리 구조**: `generate_tc.py`는 스타일·레이아웃·유효성 검사 등 xlsx 포맷만 담당. TC 내용은 `tc_data/*.py`에만 작성. 포맷 변경 시에만 엔진을 수정한다.

### tc_data 파일 형식
```python
screen_name = "화면명"   # xlsx 제목 및 파일명에 사용

tc_data = [
    # (1Depth, 2Depth, 3Depth, 4Depth, 5Depth, 6Depth, Priority, Expected Result)
    ("화면명", "시나리오", "액션", "", "", "", "P1", "기대결과가 노출된다"),
    ("", "", "다음 액션", "", "", "", "P2", "다음 기대결과가 노출된다"),
]
```

## 워크플로우

### 1. Figma 디자인 읽기 (스크린샷 우선 검증 방식)

> **원칙**: TC에 들어가는 모든 텍스트(타이틀/버튼/라벨/플레이스홀더/에러 메시지)는 **개별 프레임 스크린샷에서 눈으로 확인한 값**이어야 한다. `get_design_context`/`get_metadata`의 XML·플레이스홀더만 보고 **절대 추정하지 않는다**.

#### 1-1. URL 파싱
- `?node-id=123-456` → nodeId = `123:456` (하이픈을 콜론으로 변환)
- `figma.com/design/:fileKey/branch/:branchKey/...` → branchKey를 fileKey로 사용
- node-id가 없으면 사용자에게 요청한다

#### 1-2. 섹션/파일 단위 호출 금지
- 섹션(section) 또는 파일 전체를 `get_design_context`로 호출하면 응답이 100k+ 문자로 초과되어 실사용 불가.
- 섹션 오버뷰 스크린샷(1장에 수십 화면 축소)은 **저해상도라서 판독 불가** → 절대 판독 근거로 쓰지 않는다.
- 섹션 URL을 받았다면 먼저 `get_metadata(section nodeId)` 로 하위 프레임 목록을 조회해 개별 프레임 ID를 확보한 뒤 1-3으로 진행한다.
- `get_metadata` 도 크기 초과로 실패하면 응답이 파일로 저장되므로, `python3 -c "import json,re; ..."` 로 해당 파일을 파싱해 **화면 단위 top-level frame(width=360) 만 필터링**하여 ID와 name을 추출한다.

#### 1-3. 화면 단위 스크린샷 검증 (필수)
개별 프레임마다 다음을 수행한다:

1. `mcp__figma__get_screenshot(nodeId, fileKey)` 를 **화면 단위로 병렬 호출** (한 메시지에서 여러 프레임을 동시에 요청).
2. 응답 이미지를 직접 판독하여 다음을 verbatim으로 기록한다:
   - 화면 타이틀 / 서브카피
   - 상단 헤더 텍스트 / 진행바 상태
   - 모든 버튼의 **정확한 라벨** 과 활성/비활성 상태
   - 입력 필드의 **라벨 / 플레이스홀더 / 힌트 / 에러 메시지**
   - 체크박스/라디오 항목 텍스트 및 개수
   - 모달/바텀시트/다이얼로그의 타이틀·서브·버튼
3. 판독이 애매한 경우(저해상도·가림) 해당 프레임만 재호출하거나 사용자에게 원본 확인을 요청한다. **추정으로 채우지 않는다.**

#### 1-4. 화면 상태 variant 전부 확보
한 기능의 여러 상태(기본/입력 완료/에러/로딩/바텀시트)는 Figma에 **별도 프레임**으로 존재한다. 반드시 variant 프레임을 모두 찾아 스크린샷을 확인한다:
- 기본 / 입력 완료 / 비활성→활성 전환
- 에러 상태(중복·검증 실패·점검·네트워크)
- 바텀시트·다이얼로그·모달 노출 상태
- 이탈 확인 다이얼로그
프레임 이름에 `_입력 완료`, `_에러`, `_이탈`, `_API 불일치`, `_본문` 등 접미사가 있으면 해당 상태의 별도 캡처이므로 개별 호출 대상이다.

#### 1-5. 추정 금지 · `[확인 필요]` 표기 규칙
스크린샷에서 확인되지 않은 항목은 **어떤 경우에도 임의 값으로 채우지 않는다**. 불가피할 때는:
- TC의 Expected Result 앞에 `[확인 필요]` 를 붙이고,
- "디자인에 플레이스홀더로만 노출됨", "스크린샷 해상도 부족" 등 **이유를 함께 기재**한다.
- 디자인 자체에 이상(예: OKX 화면인데 '비트겟' 잔존 문구)이 발견되면 그대로 verbatim으로 기록하되 `[확인 필요] 기획/디자인 잔존 문구 확인` 주석을 TC에 남긴다.

#### 1-6. `get_design_context` 는 보조 수단
- 프레임 단위로 작게 호출될 때만 사용한다(섹션 호출 금지).
- XML·Code Connect 응답은 구조 파악용 참고일 뿐이며, **텍스트 확정 근거는 스크린샷**이다.
- `<instance>` 노드의 라벨은 XML에 안 나오므로 반드시 스크린샷으로 확인한다.

### 2. TC 작성 기준

| 컬럼 | 설명 |
|------|------|
| 1~6 Depth | 계층형 시나리오 (행위 중심: 동사 + 목적어) |
| Priority | P1 / P2 / P3 |
| Expected Result | 사용자 관점 ("~가 노출된다", "~으로 이동한다") |

**우선순위**
- **P1**: 화면 진입, 핵심 기능 동작, 구매 플로우, 에러 상태 처리
- **P2**: UI 노출 검증, 배지/텍스트 정확성, 스크롤
- **P3**: 안내 문구 세부 텍스트, 엣지 케이스

**커버리지 체크리스트** (화면당 필수)
- [ ] 화면 정상 진입
- [ ] 뒤로가기 / 네비게이션
- [ ] 건너뛰기(Skip) 버튼 — 온보딩·가입 플로우에서 반드시 확인
- [ ] 탭/필터 전환
- [ ] 각 카드/컴포넌트 정보 노출
- [ ] CTA 버튼 탭 → 다음 플로우 진입
- [ ] 선택형 컴포넌트(카드/탭/라디오)는 **각 항목별 개별 선택 TC** 작성 + 선택값이 다음 화면까지 유지되는지 확인
- [ ] 예외 상태 (부족, 품절, 중복 등)
- [ ] 안내 텍스트 영역

**검증 근거 원칙 (필수)**
- TC에 기록되는 **모든 텍스트는 스크린샷에서 눈으로 확인한 verbatim 값**이어야 한다.
- XML·metadata·Code Connect 응답에서 유추하거나, 일반 UX 관례로 추정해서 채우는 것을 금지한다.
- 스크린샷에서 확인되지 않은 항목은 `[확인 필요]` 표기 + 미확인 사유를 함께 남긴다.
- 작업 완료 보고 시 "추정으로 넣은 항목이 있는지"를 스스로 점검해 사용자에게 투명하게 보고한다.

## xlsx 출력 스펙

### 컬럼 구조 (A~M)

| 열 | 헤더 | 비고 |
|----|------|------|
| A~F | 1~6 Depth | |
| G | Priority | |
| H | Expected Result | |
| I | Android | 드롭다운: P / F / NA / NT |
| J | iOS | 드롭다운: P / F / NA / NT |
| K | Date | |
| L | BTS ID | |
| M | 합계 | |

- **I, J 컬럼**: 데이터 유효성 검사 드롭다운 (P / F / NA / NT), `F` 입력 시 배경색 `#FFCCCC` (연한 빨간색) 자동 적용

### 헤더 구조
- 행 1: 우상단 Priority/개수 요약 (col 18~19, 배경 `#D9D9D9`)
- 행 2: 제목 (A2:M2 병합, Arial 18pt Bold, `#002060`) + 우상단 P1 COUNTIF
- 행 3: I3:L3 병합 → "Test Result"
- 행 4: 서브헤더 (Android / iOS / Date / BTS ID), 자동 필터 A4:L4
- 행 5~: 데이터 (높이 30pt), freeze_panes = A5

### 스타일
- 폰트: Arial (제목 18pt Bold, 헤더·데이터 9pt)
- 헤더 배경: `#BFBFBF`, Bold
- 컬럼 너비: A=13.75, B=20.25, C=22.13, D=22.75, E=24.0, F=22.13, G=8.25, H=32.38, I=12.63, J=10.0, K=10.0, L=13.63, M=43.5
